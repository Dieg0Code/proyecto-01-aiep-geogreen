# Principios pedagógicos para diseñar evaluaciones

Usar estas ideas como base al diseñar evaluaciones en este repositorio.

## 1. Partir desde resultados de aprendizaje

Definir primero qué debe demostrar el estudiante:

- crear;
- explicar;
- resolver;
- diagnosticar;
- evaluar;
- o justificar.

No partir por la actividad ni por la herramienta.

## 2. Priorizar evaluación auténtica

Cuando sea posible, pedir tareas que se parezcan al trabajo real:

- construir un sitio;
- diagnosticar un problema;
- documentar una decisión;
- validar un resultado;
- o entregar un artefacto con estructura profesional.

## 3. Usar rúbrica criterial y transparente

La rúbrica debe:

- ser visible antes de la evaluación;
- usar criterios entendibles;
- mostrar pesos claros;
- y permitir comparar resultados sin arbitrariedad innecesaria.

## 4. Evaluar producto y proceso

No mirar solo el resultado final cuando el proceso también enseña oficio.

Según el caso, conviene considerar:

- calidad técnica;
- método de trabajo;
- validación;
- estructura de archivos;
- y forma de entrega.

## 5. Acotar el alcance

La evaluación debe permitir libertad, pero dentro de formatos comparables.

Evitar:

- tareas tan abiertas que no se puedan comparar;
- o tareas tan rígidas que maten criterio.

## 6. Explicitar la política de IA

Si se permite IA o agentes, dejar claro:

- qué está permitido;
- qué no;
- qué se debe revisar manualmente;
- y qué se evalúa realmente.

La IA no debe volver opaco el criterio de corrección.

## 7. Medir criterio técnico, no automatismo

Si se usa Codex o IA, la evaluación no debe premiar magia.

Debe premiar:

- capacidad de definir bien el problema;
- uso razonable de la herramienta;
- validación humana;
- y calidad final defendible.

## 8. Cerrar con entrega profesional

La forma de entrega también enseña oficio.

Cuando tenga sentido, favorecer:

- repositorio GitHub;
- estructura limpia;
- archivos entendibles;
- y entregas trazables.

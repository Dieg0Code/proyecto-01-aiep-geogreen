# Cohort Comms

Usar esta skill para escribir mensajes copy-paste-ready para estudiantes, especialmente en WhatsApp.

## Flujo de trabajo

1. Si el mensaje depende de una clase o del módulo, revisar primero `cronograma/README.md` y el `README.md` de la clase correspondiente.
2. Definir rápidamente:
   - tipo de mensaje;
   - objetivo;
   - urgencia;
   - acción esperada;
   - y tono.
3. Leer:
   - `references/tone-and-principles.md` siempre;
   - `references/message-types.md` para la estructura del mensaje;
   - `references/persuasion-patterns.md` cuando haya que mover conducta, sostener motivación o reactivar al grupo;
   - `references/audience-profile.md` cuando convenga calibrar el mensaje al perfil técnico del curso.
4. Entregar el mensaje listo para enviar, sin comentarios meta ni explicación innecesaria.

## Convención de salida

Por defecto, devolver:

- `Versión recomendada`
- `Versión más cálida`
- `Versión más directa`

Si el usuario pide una sola versión, devolver solo una.

Opcionalmente, agregar una línea breve con:

- `Objetivo`
- `CTA`
- o `Riesgo a evitar`

solo si ayuda a decidir entre versiones.

## Reglas del canal

- Escribir para estudiantes, no para el docente.
- Mantener una idea principal por mensaje.
- Dejar una acción clara cuando corresponda.
- Preferir mensajes cortos y escaneables.
- Evitar bloques largos de texto.
- Usar emojis con mucha moderación y solo si el tono lo admite.
- No usar entusiasmo falso, culpa, presión emocional ni tono de coach.
- Persuadir sin manipular.

## Reglas de tono

- Cercano, pero serio.
- Claro, pero no simplón.
- Motivador, pero sin humo.
- Técnico, pero no frío.
- Respetuoso de la inteligencia del estudiante.

## Criterios para este repo

- El público principal son estudiantes de informática de nivel técnico superior.
- Conviene reforzar claridad, pertenencia, progreso y relevancia profesional.
- Si el mensaje menciona fechas u horarios, usar información concreta.
- Si el mensaje es correctivo, ser firme sin humillar.
- Si el mensaje es de reconocimiento, elogiar conductas concretas, no adjetivos vacíos.

## Cuándo usar recursos adicionales

- `references/tone-and-principles.md`: siempre.
- `references/message-types.md`: cuando haya que elegir la estructura o el largo del mensaje.
- `references/persuasion-patterns.md`: cuando el objetivo sea activar conducta o sostener motivación.
- `references/audience-profile.md`: cuando convenga afinar el mensaje al tipo de estudiantes de este módulo.
